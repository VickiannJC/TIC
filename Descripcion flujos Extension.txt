Registro: el usuario escribe su email en el espacio de la plataforma, se detecta si el usuario lo a hecho o sino de le pide hacer (esto ya se cumple al aplastar regitrar móvil) al aplastar registrar móvil se recogen los siguientes datos: el mail del textbox, si en la tabla de subscripciones no se encuentra un registro previo de ese mismo mail, se genera un QR que tiene temporizador dándole validez de 60s, cuando se acaba este tiempo se genera un nuevo QR y se lo muestra al usuario, se detecta que el usuario ya a leído el QR y se deja de generar, este QR es leído con el celular del usuario, y se usa para emparejar el email del usuario con ese celular mediante web push, una vez escaneado el QR el usuario recibe el pedido de autorización de web push y si se autoriza se registra los datos (revisar el modelo Subscripciones, en la tabla de Subscripciones, la tabla de QRSession sirve para gestionar las regeneraciones, y la tabla Temporal para manejar los datos de la sesión. una vez guardada la información, se revisa que se haya guardado adecuadamente. Luego le aparece un mensaje HTML al usuario confirmando o denegando la subscripcion junto con un botón que le redirige a una nuava pagina web para seguir con el siguiente paso del registro, al aplastar este botón al mismo tiempo se envia al servidor de otro modulo  ( biometría), los siguientes datos un Token efimero para proteger la comunicación, el mail del usuario, y la accion que en este caso seria registro. y se espera la contestación del servidor Biometria. Si después de una hora no se registra contestación, se elimina el dato de subscripcion, temporal y QR relacionado con ese email. Si ya se obtiene respuesta se espera que devueva los siguientes datos. id_usuario, email, cadena de valores, Token de sesión(generado por biometría), api key(generado por biometría)cadena, accion:registro (NOTA: hay otras acciones dependiendo que cual sea se procede), primero se verifican los tokens y el api key, y si se comprueba se sigue con el registro, si no se notifica el error. si se comprueba positivamente, se envían los datos de id_usuario, mail, cadena de valores con un Token al modulo de psy_analizer, se espera respuesta, de ese modulo psy_analizer se espera que responda con el mail, y la confirmación del registro. y se muestra al usuario esta confirmación, si hubo un error se muestra en consola el error.
